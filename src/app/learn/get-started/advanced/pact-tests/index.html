Pact is a testing framework that supports Consumer-Driven Contract Testing between dependent systems where the integration is based on HTTP. The pattern uses contracts between interdependent services that are designed from the perspective of the consumer. The SKY UX CLI supports pact testing with the skyux pact command, which allows you to test your SPA's integration with the services that it consumes.

To implement pact testing for your SKY UX SPA, specify configuration options for pact testing in the skyuxconfig.json file, create your pact tests, and run the tests with skyux pact.

Configure pact settings

To configure your SPA to run pact tests, add the pacts property to the skyuxconfig.json file and specify configuration settings within the property.

  • To specify the consumer and provider of the service, add the consumer and provider settings. The provider is the service that exposes an API, while the consumer calls the provider's API.

    { "pacts": [ { "consumer": "{consumer}", "provider": "{provider}" } ] }

  • To specify a host and port to use for the pact testing, add the host and port settings. By default, the host is localhost and SKY UX Builder dynamically finds the next available port.

    { "pacts": [ { "consumer": "{consumer}", "provider": "{provider}" "host": "{host}", "port": "{port}" } ] }

  • You can configure several properties for the log file:

    • To specify a name for the log file, add the log setting. By default, the name is pact-{provider}.log.

    • To specify a location for the log file, add the dir setting. By default, the file is placed in the app/pacts directory.

    • To specify XXXXXXX for the log file, add the logLevel setting. By default, the XXXXXX is XXXXXXX.

    • To specify XXXXXXX for the log file, add the spec setting. By default, the XXXXX is 2.

    { "pacts": [ { "consumer": "{consumer}", "provider": "{provider}", "host": "{host}", "port": "{port}", "log": "{name}", "dir": "{directory}", "logLevel": "{XXXXXX}", "spec": "{XXXXX}" } ] }

You also need to override the provider URL to be SkyAppConfig.runtime.pactConfig.pactProxyServer/{provider}. A proxy will redirect this URL to the proper pact server. To fufill CORS, the proxy server sets the Origin header to the value that you define in the config under host.url. Otherwise, it defaults to https://host.nxt.blackbaud.com.

To run pact tests after you configure pact settings for your SPA and create the tests, open the command prompt from your SPA's directory and enter skyux pact.

SKY UX automatically configures Karma to run all pact tests in the src/app folder that match the .pact-spec.ts pattern. The command spins up a pact server for each test and also spins up a pact server for the proxy server. The proxy server takes requests from the browser that runs in the host and port specified in the configuration settings, and it also changes the CORS headers to the values that you define in the config under host.url or to the default of https://host.nxt.blackbaud.com.
